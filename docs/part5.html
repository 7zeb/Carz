Part 5: Plugin Configuration

config.yml

This is the default configuration file that will allow you to customise the Carz plugin to match your server requirements.

To make changes, edit to the value you want, save the file, then enter /carz reload into game or /carzc reload into the console. Restarting the server can apply changes also but it's very consistent, so try to use the plugin's reload command where possible.

Some properties may require the server to restart to apply the changes.

View the raw annotated configuration file here: https://pastebin.com/eQGLqpVD

This is correct as of Carz v7.0.

strings.yml

Carz will allow you to modify most of it's messages to match the language of your server, which is done through the strings.yml file.

To make changes, edit to the value you want, save the file, then enter /carz reload into game or /carzc reload into the console. The message changes should immediately apply.

If you are having problems, it may be because the yml is considered invalid if it requires ' either side of the string, when using certain characters such as %.

There are various placeholders within the translations, it is important you keep them exactly as they are i.e. %PLAYER% will be matched exactly within the plugin.

blocks.yml

This file stores the various Block Types seperate from the other config files to keep things tidy. Manual editing of this file is not encouraged unless you know what you are doing.

Addition and removal of Materials should be done through /carz add (type) and /carz remove (type) accordingly.
