Part 1: The Basics

Permissions

If you want to use permissions for the plugin, ensure Other.UsePermissions in the config.yml is set to true. Otherwise the players will not require permissions for most functionality.

Carz.*
All Carz permissions.
Carz.Admin
Ability to perform Administration commands.
Carz.Purchase
Ability to purchase an owned Car.
Carz.Place
Ability to place a Car.
Carz.Drive
Ability to drive a Car.
Carz.Upgrade
Ability to upgrade a Car.
Carz.CreateSign
Ability to create Carz signs.
Carz.BypassOwner
Bypass the ownership checks of the Car.

Commands

Every available command can be found in-game using /carz cmds. The list will be built based on permissions and configuration.

For console commands, use /carzc cmds

/carz purchase [cartype]

Request a purchase to purchase an owned car to your inventory. You will have full ownership of the car, being able to protect and lock it from other players.

You can specify which type of Car you would like to purchase by using the optional parameter, if not specified the "default" car will be used.

If Economy is enabled, a player must have sufficient funds before successfully purchasing.

* Permission: Carz.Purchase.

/carz upgrade

Request a purchase to upgrade the speed of the Car you are currently driving. This will play a sound and the car will immediately be increased in speed.

If Economy is enabled, a player must have sufficient funds before successfully purchasing.

* Permission: Carz.Upgrade.

/carz refuel

Request a purchase to fill the fuel in the current car.

No extra permission is required. If Economy is enabled a player must have sufficient funds before successfully purchasing. If configured, the cost can scale depending on the amount of fuel left in the car.

/carz fuel

Display's the current Car's fuel level. If Fuel is enabled.

If BountifulAPI is enabled, this will display as an Action Bar message.

/carz spawn [cartype]

Places an unowned Minecart in your inventory. You can specify which type of Car you would like to spawn by using the optional parameter, if not specified the "default" car will be used.

* Strict Permission: Carz.Admin.

/carz details

Display a summary of the current car's details and metadata.

If you have a Car in your hand it will display the Car's metadata.

/carz add (type) (material) [amount]

Add the specified Material into a Block Type's list. Available Block Types include: climb, speed, launch & placeable; speed and launch require the amount parameter. Changes are reflected immediately.

For more information about Block Types, click here.

* Strict Permission: Carz.Admin.

/carz remove (type) (material)

Remove the specified Material from a Block Type's list. Available Block Types include: climb, speed, launch & placeable. Changes are reflected immediately.

For more information about Block Types, click here.

* Strict Permission: Carz.Admin.

/carz stash

Stash the vehicle you are currently driving, back into a Minecart placed into your inventory. This can also be achieved by hitting the owned car.

/carz claim

Claim ownership of an un-owned Minecart you are currently in.

/carz removeowner

Remove the ownership of an owned Minecart you are currently in.

/carz createtype

Create a new Car Type. A conversation will be started to allow you to fully customise the car. Once the conversation is complete, the new Car Type is available to purchase.

If Economy is enabled, you will be asked to specify a cost of purchase.

* Strict Permission: Carz.Admin.

/carz removetype (car type)

Remove an existing Car Type from the available Car Types. The change will be applied immediately.

* Strict Permission: Carz.Admin.

/carz cartypes

List the available Car types to purchase.

* Permission: Carz.Purchase.

/carz confirm /carz cancel

If purchase confirmation is enabled, this command will be required to be entered to process the purchase request.

/carz economy

Display all economy related information, including each purchase type available.

* Strict Permission: Carz.Admin.

/carz store

Open a GUI car store, displaying each available Car type and its details. Clicking a entry will request a purchase of that Car.

* Permission: Carz.Purchase.

/carz reload

Reload the Carz configuration files.

* Strict Permission: Carz.Admin.

* = if permissions are enabled.

Strict = if permissions are disabled, op is required.
